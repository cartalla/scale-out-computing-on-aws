How it works?


Assuming your folder structure is 
└── /apps
    └── /mysoftware
        ├── /bin
        |    └── solver
        ├── /libs
        └── .....
And you submit a job using the following command:
/apps/mysoftware/bin/solver --input %INPUT_FILE% --cpus 50 --type=explicit doubleprecision --logs=mylog.txt

With the following parameters:
  • red is the name for the input parameter
  • green: is a parameter user must specify
  • blue: parameters you want to configure on behalf of the users (cannot be changed)
  • yellow: Other parameter(s) your user can set if they want to

    • Then your application profile will looks like:
  • Application Binary: /apps/mysoftware/bin/solver
  • Input parameter: --input
  • Required parameters user has to configure: --cpus
  • Required parameters configured on behalf of the user: --type=explicit doubleprecision
  • (Optional) List of scheduler parameters: scratch_size=500,instance_ami=ami-1234abcde
  • (Optional) $PATH: /apps/mysoftware/bin
  • (Optional) $LD_LIBRARY_PATH: /apps/mysoftware/libs
  • (Optional) $LD_LIBRARY_PATH: https://wiki.mycompany.com/

    • Job submitted will then looks like: 
    
#PBS -q %QUEUE_CONFIGURED_BY_USER%
#PBS -N %JOB_NAME_CONFIGURED_BY_USER%
#PBS -P %JOB_PROJECT_CONFIGURED_BY_USER%
#PBS -l scratch_size=500,instance_ami=ami-1234abcde
# (Optional) Pre Execution
# Any command specified by the user during job submission

# Configure environment
cd %PATH_WHERE_INPUT_FILE_IS_LOCATED%
export PATH=$PATH:/apps/mysoftware/bin
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/apps/mysoftware/libs

# Run command
/apps/mysoftware/bin/solver --input %INPUT_FILE_CONFIGURED_BY_USER% \
    --cpus %VALUE_CONFIGURED_BY_USER%  \
    --type=explicit doubleprecision ## FOLLOWED BY OPTIONAL PARAMETERS CONFIGURED BY USER SUCH AS --logs=mylog.txt

# (Optional) Post Execution
# Any command specified by the user post job submission
    Note: Scheduler resources are optional. If not set the job will use the value specified by default at the queue level.
